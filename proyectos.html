API ecommerce

Rest API creada con Laravel y PHP.

La API fue diseñada a partir del patrón MVC, sin embargo, la estructuré con influencias en la arquitectura limpia DDD.

Posee principios SOLID así como también patrones de diseño como Mediator y Proxy.
La API posee documentación con Postman, así como también una aplicación Front-end (hecha por mí mismo) en construcción donde se consume cada servicio.

¿Qué aprendí?


Manejar una aplicación por capas es necesario. Aumenta la seguridad, además de separar intuitivamente la lógica de negocio de la lógica funcional. Gracias también a los principios SOLID, el código se hace más elegante, legible y sobre todo mantenible a lo largo del tiempo.

Aplicación con roles y permisos

Aplicación hecha con Laravel y PHP

La aplicación fue hecha con la librería Spatie, la cual permite trabajar con roles y permisos.
Según el permiso que posea el rol asignado en el usuario, podrá hacer ciertas o todas las funcionalidades implementadas.

¿Qué aprendí?


El uso de Middleware es esencial para limitar a un usuario. Además, el agregar permisos es muy necesario en la mayoría de aplicativos debido a que no es prudente permitir a cualquier persona manejar información sensible o funcionalidad de administración.
